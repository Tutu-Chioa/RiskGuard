# 系统中尚未实现或仅为占位的功能清单

以下为当前代码与交互下「未实现」或「占位/假数据」的功能，便于你决定是否要补齐实现。

---

## 1. 风险警报页（首页 → 风险警报 / 侧栏「风险警报」）

- **现状**：页面内三张卡片（高风险、中风险、信息提示）为**写死文案**：
  - 「ABC科技有限公司风险评分升至85分，请立即关注」
  - 「XYZ制造有限公司出现新的法律纠纷」
  - 「创新软件有限公司完成新一轮融资」
- **缺失**：未调用后端 `GET /api/alerts`，也未根据 `risk_alerts` 表或企业真实数据渲染列表；「处理警报」/「查看详情」固定跳转企业 ID 1/2/3，未与警报关联的企业 ID 绑定。
- **后端**：已有 `risk_alerts` 表及 `GET /api/alerts` 接口，可在此基础上做列表与详情联动。

---

## 2. 系统设置 - 用户设置（姓名、邮箱、警报阈值）

- **现状**：设置页存在「用户设置」区块（姓名、邮箱、警报阈值、保存按钮）。通用「保存设置」会调用 `PUT /api/auth/me` 并写入本地。
- **可能问题**：后端为 `PATCH /api/auth/me`（update_profile），若只接受 PATCH 则 PUT 可能失败；且后端 update_profile 是否持久化「警报阈值」等字段需确认。若未持久化，则警报阈值仅为前端/本地状态。

---

## 3. 两因素认证、数据备份（设置 → 安全）

- **现状**：设置中有「两因素认证」「数据备份」等开关，并随「保存设置」一起提交到服务器。
- **可能问题**：后端是否存储并真正实现 2FA、备份逻辑未在本次排查中确认；若未实现，则仅为前端状态，无实际安全/备份效果。

---

## 4. 忘记密码

- **现状**：登录/注册页无「忘记密码」入口与流程。
- **缺失**：无找回密码、重置密码的界面与接口。

---

## 5. 风险警报的自动创建与推送

- **现状**：企业详情页有「风险警报」区块，数据来自 `company.alerts`（后端 get_company 会带出 risk_alerts）；列表展示依赖后端是否写入 `risk_alerts`。
- **可能问题**：若系统从未在业务逻辑中「自动创建」风险警报（例如根据风险等级变化、新闻负面度等写入 `risk_alerts`），则警报列表可能始终为空，需确认是否有定时任务或事件驱动写入。

---

## 6. 设置页「恢复默认」与主题键一致

- **现状**：恢复默认时 `setThemeColor('purple')`，与主题预设键一致即可。
- **说明**：若后续主题键名调整，需同步修改恢复默认逻辑。

---

## 7. 其他已对接或基本实现的功能（供对照）

- **个人中心**：有 `/api/auth/me`、`PATCH /api/auth/me` 及修改密码等，已与后端对接。
- **企业对比**：使用 `GET /api/companies` 与 `GET /api/companies/:id`，已对接。
- **上传资料**：上传与文档分析流程存在，具体分析逻辑与入库取决于后端实现。
- **任务状态、数据分析、企业管理、企业详情、三灯状态**：均与现有后端接口对接，使用真实或聚合数据。
